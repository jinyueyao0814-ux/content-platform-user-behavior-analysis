# 内容平台用户行为与商业化转化分析

## 项目背景

模拟一个内容平台（涵盖短视频与旅游电商场景）的业务数据，使用 MySQL 完成用户增长、内容消费、转化漏斗、用户分层与 A/B 测试五个维度的分析。

## 数据说明

共 6 张表，模拟平台核心业务数据：

| 表名 | 说明 |
|------|------|
| users | 用户基础信息，含注册日期、城市、年龄段、设备 |
| videos | 视频内容库，含分类、时长、发布日期 |
| user_actions | 用户行为记录，含观看时长、点赞、评论、分享 |
| travel_orders | 旅游订单数据，含目的地、产品类型、订单状态、GMV |
| funnel_events | 转化漏斗埋点，记录用户从浏览到下单的每个步骤 |
| ab_test | A/B 测试数据，记录实验组与对照组的点击与转化行为 |

## 分析模块

### 模块一：用户活跃度与留存分析
**业务问题：** 平台每月活跃用户趋势如何？新用户次月留存率是多少？

**主要结论：**
- MAU 从 2 月持续增长至 5 月，平台处于增长阶段
- 1 月注册用户次月留存率最高，5 月因数据截止排除在外

**涉及技术：** DATE_FORMAT、COUNT DISTINCT、LEFT JOIN、子查询、CASE WHEN 打标

---

### 模块二：内容消费行为分析
**业务问题：** 各内容分类的完播率和互动率有何差异？

**主要结论：**
- 知识类完播率最高（94%），用户目的性强
- 美食类互动率最低（56%），内容同质化，缺乏互动点
- 旅游类两项指标均处于中等水平，有提升空间

**涉及技术：** 多表 JOIN、AVG、SUM/COUNT 打标聚合

---

### 模块三：旅游产品转化漏斗分析
**业务问题：** 用户从浏览产品到最终下单，每一步的转化率是多少？

**主要结论：**
- view_product → add_to_cart 流失最多，转化率 73%，详情页吸引力有待提升
- start_checkout → paid 转化率最高（75%），到达结算页的用户购买意愿强
- 优化重点应放在详情页到加购环节

**涉及技术：** LAG 窗口函数、CASE WHEN 自定义排序、多层子查询

---

### 模块四：用户 RFM 分层
**业务问题：** 如何根据用户的消费行为对其进行价值分层？

**主要结论：**
- 高价值用户（1人）：最近下单、消费金额高，重点维护，推送高端定制产品
- 中价值用户（6人）：重点转化，提升复购频次
- 低价值用户（3人）：距上次购买时间长，通过优惠券做召回

**涉及技术：** DATEDIFF、多层子查询、CASE WHEN 打分、用户分层模型

---

### 模块五：A/B 测试分析
**业务问题：** 旅游产品详情页新版设计是否优于旧版？

**主要结论：**
- B 组（新版）点击率 73% vs A 组 67%，提升 6 个百分点
- B 组点击后转化率 73% vs A 组 50%，提升显著
- 新版详情页在点击和转化两个环节均有提升，建议全量上线
- 注：样本量较小，实际决策需结合卡方检验确认统计显著性

**涉及技术：** WHERE 条件过滤、分组聚合、HAVING vs WHERE 区分

---

## 技术栈

- **数据库：** MySQL 8.0
- **核心技术：** 窗口函数（LAG）、多层子查询、多表 JOIN、日期函数、CASE WHEN、聚合函数
- **分析模型：** 漏斗分析、RFM 用户分层、A/B 测试
